虽然可以直接在 HTMl 文件中嵌入 JavaScript 代码，但通常
            认为最佳事件是尽可能将 JavaScript 代码放在外部文件中。不过这个
            最佳实践并不是明确的强制性规则。推荐使用外部文件的理由如下：
                · 可维护性。JavaScript 代码分散到很多 HTMl 中，会导致维护困难。
            而用一个目录保存所有 javascript 文件，就更容易维护，这样开发者就可以
            独立于使用它们的 HTML 页面来编辑代码。
                · 缓存。浏览器根据特定设置缓存所有外部连接的 JavaScript 文件，
            这意味着如果两个页面都用到同一个文件，则文件只需要下载一次，最终
            页面加载也会更快。
                · 适应未来。放到外部文件，就不必考虑 XHTML 或前面提到的废弃语法。
